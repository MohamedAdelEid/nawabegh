Type 2
📝 ShortQuiz
فلو بسيط — السيرفر هو اللي بيكمّل بعد submit ناجح
Student Flow
STEP 01
تحميل المحاولة
جيب الأسئلة والإجابات اللي اتحفظت قبل كده
GET /api/v1/progress/quiz-stations/{stationId}/attempt
STEP 02
Autosave لكل إجابة
كل ما الطالب يختار option → ابعت على طول
POST /api/v1/progress/quiz-stations/{stationId}/answers
body: { "questionId": "…", "selectedOptionId": "…" }
STEP 03
Submit Quiz
الطالب يضغط "تسليم" — السيرفر يحسب النتيجة من الإجابات المحفوظة
POST /api/v1/progress/quiz-stations/{stationId}/submit
⚠ body فاضي — ما تبعتش scoreAchieved
STEP 04
اكتمال تلقائي
لو نتيجة% ≥ Quiz.passScore → السيرفر بيكمّل المحطة داخليًا بدون طلب منك
✓ لا تبعت /complete يدوي أبدًا
✓ بعدها refetch stations/progress
🚫 ما تعملهوش
ما تبعتش scoreAchieved في أي body — السيرفر بيتجاهله
ما تبعتش POST /progress/stations/{id}/complete يدوي — مش محتاجه
ما تحتسبش النتيجة في الـ client — خليها للسيرفر
✅ سيناريوهات الـ Response
نتيجة ≥ passScore → 200 + station Completed + نقاط
نتيجة < passScore → 400 + رسالة بالفرق (passScore vs achieved)
ما فيش attempt مسلّم → 400 "No submitted attempt"
⚙️ Admin Config
completionThreshold على المحطة = null دايمًا
العتبة بتتحدد في Quiz.passScore (نسبة %)
أسئلة Quiz = بيتبعوا QuizzesController
Type 0
📡 LiveStream
محطة واحدة بتشيل البث المباشر والتسجيل — branch على runtimeMode
Phase 1 — البث المباشر
STEP 01
فحص الحالة
اقرأ runtimeMode وحالة الطالب الحالية
GET /api/v1/live-stations/{stationId}/info
runtimeMode: Upcoming | Live | Recorded | EndedWithoutRecording
STEP 02
انضمام للبث
بس لو runtimeMode = Live وحالة الطالب ≠ Locked
POST /api/v1/live-stations/{stationId}/join
⚠ ما تفتحش لو status = Locked حتى لو وقت البث جه
STEP 03
حضور Zoom (تلقائي)
السيرفر بيحسب دقائق الحضور من Zoom Webhooks تلقائيًا — ما عندكش دخل فيه
⚠ لا تبعت attended: true في أي body — بيتتجاهل
STEP 04
بعد انتهاء البث
السيرفر بيحدد الحالة تلقائيًا بعد ما البث ينتهي
حضر ≥ threshold دقائق → Completed
حضر < threshold → Incomplete
ما حضرش خالص → Missed
Phase 2 — إكمال بالتسجيل (Incomplete أو Missed)
STEP 01
تحقق من runtimeMode
لازم يكون "Recorded" عشان تعرض التسجيل — لو EndedWithoutRecording ما تعرضش
GET /api/v1/live-stations/{stationId}/info
راجع كمان recordingUrl وstudentProgressStatus
STEP 02
مشاهدة التسجيل
الطالب يفتح recordingUrl — تتبع نسبة المشاهدة في الـ client
UI: "أكمل بالتسجيل" (Incomplete) | "تابع بالتسجيل" (Missed)
STEP 03
إرسال نسبة المشاهدة
لما تصل للـ threshold أو أكتر — ابعت الـ request
POST /api/v1/progress/stations/{stationId}/complete
{ "percentageCompleted": 80 }
STEP 04
Completed
المحطة تنتقل Incomplete/Missed → Completed
✓ نقاط تُعطى مرة واحدة بس (idempotent)
✓ اللي بعدها بتفتح لو autoUnlock = true
حالات التقدم — station progress statuses
🔒 Locked — المحطات السابقة ما كملتش
🔵 Available — متاحة للانضمام
🟡 InProgress — انضم للبث
⚠️ Incomplete — حضر بس ناقصه → أكمل بالتسجيل
❌ Missed — ما حضرش → تابع بالتسجيل
✅ Completed — خلص
🚫 ما تعملهوش
ما تبعتش attended: true — الـ server بيتجاهله
ما تعملش screen أو StationType منفصل للتسجيل — نفس المحطة
ما تفتحش محطة Locked حتى لو وقتها جه
⚙️ Admin Config
completionThreshold = نسبة الدقائق المطلوبة (0–100)
نفس الـ threshold بيُستخدم للـ live (دقائق Zoom) وللتسجيل (watch %)
الجلسة بتُنشأ بـ POST /live-sessions وتنتهي بـ POST /live-sessions/{id}/end
📺 runtimeMode Logic
Upcoming → عرض countdown، ما تبينش join
Live → بيّن join button
Recorded → بيّن تسجيل + complete flow
EndedWithoutRecording → بيّن رسالة اعتذار بس

Student Flow
STEP 01
تحميل الـ Deck
جيب الكروت كلها للمحطة دي
GET /api/v1/progress/flashcard-stations/{stationId}/deck
STEP 02
مراجعة الكروت
الطالب يمشي على الكروت في الـ UI — انت بتحسب كام % راجع
track progress locally في الـ client
STEP 03
إرسال النسبة
لما يوصل للـ threshold أو أكتر — ابعت complete
POST /api/v1/progress/stations/{stationId}/complete

اكتمال

Type 3
⚔️ Challenge
الأكثر تعقيدًا — بيجمع HTTP + SignalR في نفس الفلو
Student Flow — 6 خطوات
STEP 01
فحص التفاصيل
اعرض معلومات التحدي — تحقق إن المحطة مش Completed قبل ما تعرض "العب"
GET /api/v1/student/challenges/{challengeId}
⚠ لو Completed → اخفي زر "العب تاني"
STEP 02
وصّل SignalR Hub
اتصل بالـ Hub بنفس JWT المستخدم في HTTP — جيب connectionId
/hubs/challenge
Auth: Bearer token (Student role)
STEP 03
دخول الطابور
ابعت connectionId اللي جبته من الـ Hub للسيرفر للـ matchmaking
POST /api/v1/student/challenges/{challengeId}/queue
{ "signalRConnectionId": "…" }
⚠ 409 = المحطة Completed بالفعل
STEP 04
استنى المباراة
السيرفر بيبعت sessionId عبر SignalR لما يلاقيلك خصم
استدعي: RegisterSession(sessionId)
بيضيف الـ connection لـ group "challenge-session:{id}"
STEP 05
إجابة الأسئلة
لكل سؤال ابعت إجابة — بتجي الأسئلة من GET session/questions
GET /student/challenges/sessions/{sessionId}/questions
POST /student/challenges/sessions/{sessionId}/answers
{ "questionId": "…", "optionId": "…" }
STEP 06
إنهاء الجلسة
بعد الأسئلة كلها — ابعت finish بـ body فاضي
POST /student/challenges/sessions/{sessionId}/finish
body: {}
✓ الاتنين لاعبين بيكملوا المحطة تلقائيًا
✓ اللي بعدها بتفتح للاتنين

🏆 نظام النقاط
الفايز = 100% من station.pointsReward
الخسران = 50% من station.pointsReward
تعادل = 50% للاتنين
الفوز والخسارة ما بيوقفوش الاكتمال — الاتنين بيكملوا
⚙️ Admin Config
completionThreshold = null دايمًا
pointsReward على المحطة هو اللي بيتقسم
التحدي بيتبنى بـ POST /challenges مع scheduleDate وtimeZoneId
AI questions: POST /challenges/{id}/generate-questions
🚫 ما تعملهوش
ما تعرضش "العب تاني" لو station Completed — 409 هيجيلك
ما تفرقش في الـ UX بين فايز وخسران للـ station unlock
ما تنسى تبعت connectionId في الـ queue — matchmaking بيحتاجه

📎 HelperResource
أبسط محطة في النظام — جيب المحتوى واعرضه وكمّل
Student Flow
STEP 01
تحميل المحطة
جيب الـ payload — بيشمل الملفات والروابط المرتبطة بالمحطة
GET /api/v1/Station/{stationId}
STEP 02
الطالب يشوف المحتوى
افتح الملفات أو الروابط للطالب — ما في tracking للمشاهدة
attachments + links في الـ payload
STEP 03
Complete يدوي
بعت complete بـ body فاضي — مفيش threshold ولا score
POST /api/v1/progress/stations/{stationId}/complete
body: { }
✓ Completed فورًا
STEP 04
فتح التالي
لو autoUnlockOnPreviousComplete = true → المحطة التالية بتفتح
refetch stations/progress لتحديث الـ UI
⚙️ خصائص المحطة
completionThreshold = null دايمًا
مفيش score ولا percentage
المحطة دي مش بتأثر على path completion حساب عادةً
🎯 متى تبعت complete؟
الـ spec بيقول "auto on open" — لكن الـ backend بيكمّل بس لما تبعت الـ endpoint
يعني ابعته بعد ما الطالب يشوف المحتوى أو يضغط "تمام"

🔒 Sequential Unlock — قاعدة ذهبية
محطة Locked = ما تفتحهاش ولا تنضملها حتى لو وقتها جه
بعد اكتمال محطة + autoUnlock = true → التالية بتبقى Available
ما في "unlock next" API للطالب — بس refetch /stations/progress
🔁 Idempotency
re-complete محطة Completed → 200 بدون نقاط جديدة
initialize مرتين → 409 Conflict = طبيعي، تعامله كـ success
Challenge queue على Completed → 409 = اخفي "العب تاني"
🎯 Path Completion
لما كل المحطات non-helper بتكمل → pathCompleted: true
Path completion = 0 bonus points (مش في نقاط إضافية)
pathPointsEarned في الـ response بيبقى null
🛡️ Admin Override
POST /admin/stations/{id}/complete-override (Admin only)
بيشيل Locked → Available → Completed تلقائيًا
reason مطلوب (max 500 chars) — بيتسجل في audit log
